## Rules

- Fixtures contain no secrets, customer data, mutable remote URLs required for CI, or raw proprietary page bodies.
- Record external responses once, sanitize them, retain headers/status needed for realistic adapters, and document origin/version.
- Frontend story/test fixtures import canonical payloads through the contract package; no hand-written divergent shape.
- Live service smoke tests never rewrite canonical fixtures automatically.
- A contract fixture is updated only alongside validators and all impacted tests.
- Delete stale fixtures when their consumer disappears; fixture count is not a quality metric.
